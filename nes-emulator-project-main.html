RetroDS is a self-designed, battery-powered handheld gaming console built from scratch using the ESP32-WROOM-32E microcontroller. The device emulates NES / retro games using a ported emulation engine, outputs stereo-quality audio via a PCM5102 I2S DAC, renders to a 240×240 SPI TFT display, and accepts player input through a custom-laid 8-button interface.

All hardware — from breadboard prototype through to hand-soldered perfboard — was designed, assembled, and debugged in-house at the University of Western Australia. The front panel board carries the display, D-pad, and action buttons. The back board carries the ESP32, audio module, MicroSD breakout, and power management IC.

The project demonstrates full-stack embedded systems development: hardware architecture, subsystem integration, firmware written in C++ on the Arduino-ESP32 framework, FAT32 ROM loading, I2S DMA audio management, and battery-backed power design.

Firmware Architecture

The firmware is written in C++ using the Arduino-ESP32 framework. The NES emulation engine runs on Core 0 while I2S audio DMA and display SPI transfers are managed with careful timing to avoid frame tears and audio dropouts.

NES Emulation Core
Full 6502 CPU emulation with cycle-accurate timing and PPU rendering
Implements a full 6502 CPU emulator with cycle-accurate timing. The PPU (Picture Processing Unit) renders to a 240×240 frame buffer. NES ROMs are loaded from MicroSD via FAT32 filesystem (up to 512 kB). SPI Flash memory-mapped ROM access via the esp_spi_flash mmap API provides faster read throughput without copying ROM data into DRAM.
Display Subsystem
ST7789 driven over hardware SPI at 40 MHz with double-buffer DMA
The ST7789 1.3" TFT is driven over hardware SPI at 40 MHz. A double-buffer strategy builds each frame in RAM and pushes it via DMA, preventing screen tearing. A custom init sequence corrects colour ordering (BGR swap). Backlight brightness is controlled via the LEDC PWM peripheral.
Audio Subsystem
PCM5102 I2S DAC — 16-bit stereo at 44.1 kHz with DMA ring buffer
The PCM5102 I2S DAC module outputs 16-bit stereo audio at 44.1 kHz. An I2S DMA ring buffer keeps audio smooth and independent of CPU emulation load. The audio task runs at higher FreeRTOS priority than the PPU rendering loop to prevent dropouts under heavy emulation load.
Input Handling
8 GPIO pins with internal pull-ups, polled every emulation frame
Eight GPIO pins use internal pull-up resistors. Inputs are polled every emulation frame (~16 ms), eliminating the need for interrupt-driven edge detection. Button reads are debounced by sampling over consecutive frames. The 8-bit input state maps directly to the NES controller byte format expected by the emulator core.

Build Progression

The project followed a structured hardware bring-up sequence: component sourcing → breadboard prototype → subsystem integration → perfboard assembly → power system integration.

Phase 1 — Component Sourcing
ST7789 display, PCM5102 audio module, MicroSD breakout, tactile buttons
Sourced all modules independently: ST7789 1.3" TFT display, PCM5102 I2S DAC breakout, MicroSD module, 8× tactile buttons, IP5306 power bank IC, and LiPo cell. All components were bench-tested for basic electrical continuity and voltage compatibility before integration.
Phase 2 — Breadboard Prototype
First boot — display confirmed, SD card detected, subsystems validated
Built the first breadboard prototype with ESP32-WROOM-32E, ST7789 display, MicroSD module, PCM5102 audio module, and button matrix. Achieved first boot with display showing a test screen and SD card detected at 30 GB. Audio output confirmed at 44.1 kHz. Button GPIO reads validated with serial monitor. SPI bus conflict between display and SD card was identified and resolved at this stage.
Phase 3 — Perfboard Assembly & Soldering
Hand-soldered front and back boards with colour-coded wiring harness
All components transferred from breadboard to perfboard using a temperature-controlled soldering station with paste flux. The front panel board carries the ST7789 display, D-pad buttons, and A/B/Start/Select buttons. The back board carries the ESP32, PCM5102 audio module, MicroSD breakout, and IP5306 power management IC. A colour-coded wiring harness connects the two boards.
Phase 4 — Power Integration & Final Testing
LiPo cell integrated with USB-C charge, boost converter, and 3.3 V regulation
Integrated the LiPo 3.7 V battery with TP4056 USB-C charge IC and IP5306 5 V boost converter. Logic voltage regulated to 3.3 V for all digital subsystems. Full NES ROM loading from MicroSD confirmed functional. Audio playback and display rendering verified. Prototype declared complete and functional with all subsystems operational.

Engineering Challenges

SPI Bus Conflict — Display vs MicroSD
Both the ST7789 display and MicroSD module use SPI. The SD card was moved to software SPI (bit-banged) on separate GPIO pins (MOSI GPIO17, MISO GPIO13, SCK GPIO14), freeing the hardware SPI bus exclusively for the display. This maintained 40 MHz display throughput without bus arbitration overhead or frame rate loss.
I2S Audio Dropouts at High Emulation Load
Initial testing showed audio glitches when the CPU was under heavy NES emulation load. Resolved by pre-filling the I2S DMA ring buffer at startup and assigning the audio task a higher FreeRTOS priority than the PPU rendering loop. This ensured the DMA buffer never ran dry regardless of emulation CPU demand.
GPIO Input Noise on High-Impedance Button Lines
Floating inputs caused spurious button presses during testing. Fixed by enabling internal GPIO pull-up resistors in firmware and debouncing button reads by sampling over consecutive emulation frames rather than using interrupt-driven edge detection — eliminating false triggers without requiring external hardware.
SPI Flash ROM Mapping for Large ROMs
ROMs larger than available DRAM could not be copied into RAM for emulation. Solved using the esp_spi_flash mmap API to memory-map ROM data directly from SPI flash, allowing the emulator to access ROM content as a const uint8_t* pointer without occupying DRAM. Enabled support for ROMs up to 512 kB.

Next Steps

Custom PCB Design — EasyEDA / JLCPCB
Full schematic capture in EasyEDA targeting a 2-layer PCB manufactured via JLCPCB. The PCB will integrate all modules onto a single board with proper ground plane, power rails, and a dedicated connector for the LiPo cell. Eliminates point-to-point perfboard wiring and reduces assembly time.
3D-Printed Enclosure — Fusion 360
Design a snap-fit shell in Fusion 360 around the final PCB footprint, with screen cutout, button holes aligned to D-pad and action buttons, USB-C port cutout, and headphone/speaker outlet. Printed in PLA on Creality Ender-3 S1 Pro.
Speaker & Hardware Volume Control
Add a 0.5 W 8Ω speaker for built-in audio output and a hardware volume control potentiometer wired to an analog GPIO. The PCM5102 DAC supports hardware volume attenuation via its SCK/BCK lines, enabling software volume control as a secondary option.
ROM Selection Menu UI
Build an on-device ROM selection menu rendered on the ST7789 display, reading available .nes files from MicroSD and presenting a scrollable list. Navigation via D-pad with A to confirm. This replaces the current method of hardcoding a single ROM filename in firmware.
